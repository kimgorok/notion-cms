# PRD: 개인 개발 블로그 (Notion CMS 기반)

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **프로젝트명** | 개인 개발 블로그 |
| **목적** | Notion을 CMS로 활용한 개인 Claude 활용 기술 블로그 |
| **대상 사용자** | 개발자, Claude AI 활용에 관심 있는 독자 |
| **배포 환경** | Vercel |

### 1.1 CMS로 Notion을 선택한 이유

- Notion에서 글을 작성하면 **별도 배포 없이 자동으로 블로그에 반영**
- 친숙한 Notion 편집기로 콘텐츠 관리 가능
- Notion 데이터베이스의 필터/정렬 기능을 CMS 속성으로 활용
- 별도 백엔드 서버 불필요 (서버리스 아키텍처)

## 3. Notion 데이터베이스 구조

블로그 콘텐츠를 관리하는 Notion 데이터베이스의 속성 정의입니다.

| 속성명 | Notion 타입 | 설명 | 예시 값 |
|--------|------------|------|---------|
| **Title** | `title` | 글 제목 | "Claude로 코드 리뷰 자동화하기" |
| **Category** | `select` | 글 카테고리 (단일 선택) | Claude, Next.js, TypeScript |
| **Tags** | `multi_select` | 태그 (복수 선택) | claude-api, automation, tips |
| **Published** | `date` | 발행일 | 2026-05-31 |
| **Status** | `select` | 글 상태 | `초안` / `발행됨` |
| **Content** | page content | 글 본문 (Notion 페이지 내용) | Notion 블록 구조 |

> **Status가 `발행됨`인 글만** 블로그에 노출됩니다.

## 4. 주요 기능

### 4.1 기능 목록

| 기능 | 우선순위 | MVP 포함 |
|------|---------|---------|
| Notion 데이터베이스에서 글 목록 가져오기 | 필수 | ✅ |
| 개별 글 상세 페이지 표시 | 필수 | ✅ |
| 반응형 디자인 | 필수 | ✅ |
| 카테고리별 필터링 | 중요 | ✅ |
| 검색 기능 | 중요 | ✅ |
| 다크 모드 | 선택 | ❌ |
| RSS 피드 | 선택 | ❌ |

### 4.2 기능 상세

#### F-01. 글 목록 조회
- Notion API를 통해 `Status = 발행됨`인 글만 조회
- 발행일(`Published`) 기준 내림차순 정렬
- 글 카드에 표시: 제목, 카테고리, 태그, 발행일, 요약(본문 첫 문단)

#### F-02. 글 상세 페이지
- Notion 페이지 블록을 HTML로 변환하여 렌더링
- 지원 블록 타입: 단락, 제목(H1~H3), 코드 블록, 인용, 목록, 이미지, 구분선
- 메타데이터 표시: 카테고리, 태그, 발행일

#### F-03. 카테고리 필터링
- 전체 카테고리 목록을 동적으로 생성
- 카테고리 선택 시 해당 글 목록만 표시
- URL 쿼리 파라미터로 필터 상태 유지 (`?category=Claude`)

#### F-04. 검색
- 글 제목 기준 클라이언트 사이드 검색
- 검색어 입력 시 실시간 필터링

#### F-05. 반응형 디자인
- 모바일(~767px), 태블릿(768~1023px), 데스크톱(1024px~) 대응
- 글 목록: 모바일 1열 → 태블릿 2열 → 데스크톱 3열

### Phase 2. Notion 데이터베이스 구성

- [ ] 속성 정의 (Title, Category, Tags, Published, Status)
- [ ] 샘플 글 3~5개 작성

### Phase 3. Notion API 연동 레이어
- [ ] `lib/notion.ts`: Notion 클라이언트 초기화 및 유틸 함수 구현

  - `getPost(id)`: 단일 글 조회
  - `getBlocks(id)`: 페이지 블록(본문) 조회

### Phase 4. 글 목록 페이지 구현

### Phase 5. 글 상세 페이지 구현
- [ ] `app/posts/[slug]/page.tsx`: 글 상세 페이지

- [ ] ISR(Incremental Static Regeneration) 설정으로 성능 최적화

### Phase 6. 스타일링 및 마무리

## 7. API 설계

### 환경 변수

```env
NOTION_API_KEY=secret_xxxxxxxxxxxx
NOTION_DATABASE_ID=xxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

### Notion API 호출 구조

```
lib/
└── notion.ts
    ├── notionClient          # @notionhq/client 인스턴스
    ├── getPosts()            # 글 목록 조회 (Status=발행됨, 날짜 내림차순)
    ├── getPost(pageId)       # 단일 글 메타데이터 조회
    └── getPageBlocks(pageId) # 글 본문 블록 조회
```

## 8. 비기능 요구사항

| 항목 | 요구사항 |
|------|---------|
| **성능** | ISR 적용으로 글 상세 페이지 첫 로딩 1초 이내 |
| **SEO** | Next.js Metadata API로 글별 og:title, og:description 설정 |
| **접근성** | 시맨틱 HTML 태그 사용, 이미지 alt 속성 필수 |
| **보안** | Notion API 키는 서버 사이드에서만 사용 (클라이언트 노출 금지) |
| **캐시** | ISR revalidate 60초 설정으로 Notion 변경사항 반영 |
